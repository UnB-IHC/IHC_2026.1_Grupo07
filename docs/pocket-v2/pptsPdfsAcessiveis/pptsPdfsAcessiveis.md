# PPTs e PDFs Acessíveis

> **[NOVO - TG10/Grupo07]** Esta seção apresenta boas práticas para criação de apresentações (PowerPoint, Google Slides e similares) e documentos PDF acessíveis. As recomendações são baseadas na **WCAG 2.2**, na **ABNT NBR 17225:2025** e em guias de acessibilidade digital.

Apresentações e documentos digitais devem ser produzidos de forma que possam ser compreendidos por todos os usuários, incluindo aqueles que utilizam leitores de tela, navegação por teclado, ampliadores de tela e outras tecnologias assistivas.

## Estrutura do Documento

* [ ] Utilizar títulos e subtítulos hierárquicos.
* [ ] Utilizar os estilos nativos da ferramenta para títulos.
* [ ] Evitar criar títulos apenas aumentando o tamanho da fonte.
* [ ] Manter organização lógica das informações.
* [ ] Dividir conteúdos extensos em seções menores.
* [ ] Garantir consistência visual entre páginas ou slides.

## Títulos

* [ ] Todo documento deve possuir um título principal.
* [ ] Utilizar hierarquia lógica de títulos.
* [ ] Evitar pular níveis de títulos.
* [ ] Utilizar títulos descritivos e objetivos.
* [ ] Garantir que os títulos representem corretamente o conteúdo da seção.

### Exemplo

✅ 1. Introdução

✅ 1.1 Objetivos

✅ 1.2 Escopo

❌ Introdução → Resultado → 1.2 Escopo

## Texto e Legibilidade

* [ ] Utilizar linguagem simples e objetiva.
* [ ] Evitar blocos extensos de texto.
* [ ] Utilizar tamanho de fonte adequado.
* [ ] Garantir espaçamento adequado entre linhas.
* [ ] Evitar alinhamento justificado.
* [ ] Evitar excesso de texto em um único slide.
* [ ] Utilizar listas quando apropriado.

## Contraste

* [ ] Garantir contraste adequado entre texto e plano de fundo.
* [ ] Evitar texto claro sobre fundo claro.
* [ ] Evitar texto escuro sobre fundo escuro.
* [ ] Verificar contraste de gráficos e elementos visuais.
* [ ] Garantir contraste adequado para links e botões.
* [ ] Evitar depender exclusivamente da cor para transmitir informações.

## Imagens

* [ ] Utilizar imagens relevantes ao conteúdo.
* [ ] Evitar imagens que contenham texto essencial.
* [ ] Garantir boa qualidade visual das imagens.
* [ ] Evitar excesso de elementos decorativos.

## Texto Alternativo (Alt Text)

* [ ] Toda imagem informativa deve possuir texto alternativo.
* [ ] O texto alternativo deve transmitir a informação relevante da imagem.
* [ ] Evitar descrições genéricas como "imagem" ou "foto".
* [ ] Imagens decorativas devem possuir texto alternativo vazio.
* [ ] Diagramas complexos devem possuir descrição complementar.

### Exemplo

Imagem:

Fluxograma do processo de autenticação.

Texto alternativo:

"Fluxograma demonstrando as etapas de login, validação de credenciais e acesso ao sistema."

## Gráficos e Diagramas

* [ ] Fornecer descrição textual dos dados apresentados.
* [ ] Não depender exclusivamente de cores para diferenciar informações.
* [ ] Identificar claramente legendas e rótulos.
* [ ] Garantir contraste adequado entre elementos do gráfico.
* [ ] Disponibilizar resumo textual dos principais resultados.

## Tabelas

* [ ] Utilizar cabeçalhos de linha e coluna.
* [ ] Evitar células mescladas quando possível.
* [ ] Manter estrutura simples.
* [ ] Garantir leitura lógica dos dados.
* [ ] Não utilizar tabelas apenas para layout.

## Ordem de Leitura

* [ ] Verificar a ordem de leitura dos elementos.
* [ ] Garantir que a sequência corresponda à ordem visual.
* [ ] Validar a navegação utilizando leitor de tela.
* [ ] Garantir que caixas de texto sejam lidas corretamente.

### Importância

A ordem de leitura incorreta pode fazer com que leitores de tela interpretem conteúdos fora de contexto, dificultando a compreensão do documento.

## Hiperlinks

* [ ] Utilizar textos descritivos para links.
* [ ] Evitar "Clique aqui".
* [ ] Informar quando um link realizar download.
* [ ] Garantir que o destino seja compreensível apenas pelo texto do link.

### Exemplo

❌ Clique aqui

✅ Baixar Relatório de Acessibilidade

## Elementos Multimídia

* [ ] Vídeos devem possuir legendas.
* [ ] Áudios devem possuir transcrição.
* [ ] Conteúdos importantes não devem depender exclusivamente de áudio.
* [ ] Conteúdos importantes não devem depender exclusivamente de elementos visuais.

## PDFs

Antes de exportar para PDF:

* [ ] Definir idioma do documento.
* [ ] Inserir título do documento nos metadados.
* [ ] Garantir estrutura correta de títulos.
* [ ] Verificar textos alternativos.
* [ ] Validar a ordem de leitura.
* [ ] Verificar contraste de cores.
* [ ] Garantir que o PDF esteja marcado (tagged PDF).
* [ ] Executar ferramenta de verificação de acessibilidade.

## PowerPoint

Antes de compartilhar uma apresentação:

* [ ] Utilizar layouts nativos dos slides.
* [ ] Verificar ordem de leitura dos objetos.
* [ ] Inserir texto alternativo em imagens.
* [ ] Garantir contraste adequado.
* [ ] Evitar excesso de animações.
* [ ] Validar acessibilidade com a ferramenta nativa do PowerPoint.

## Checklist Final

### Estrutura

* [ ] Hierarquia de títulos correta.
* [ ] Organização lógica do conteúdo.
* [ ] Linguagem clara e objetiva.

### Imagens

* [ ] Texto alternativo adicionado.
* [ ] Imagens decorativas identificadas corretamente.
* [ ] Gráficos descritos textualmente.

### Acessibilidade

* [ ] Contraste adequado.
* [ ] Ordem de leitura validada.
* [ ] Links descritivos.
* [ ] Navegação compatível com leitores de tela.

### Exportação

* [ ] Documento marcado para acessibilidade.
* [ ] PDF validado.
* [ ] Metadados preenchidos.
* [ ] Idioma definido.
