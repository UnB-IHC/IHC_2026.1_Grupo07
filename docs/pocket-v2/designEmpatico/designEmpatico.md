# :material-heart-outline: Design Empático

!!! info "[NOVO - TG10/Grupo07]"
    Esta seção foi adicionada para complementar as diretrizes de acessibilidade do projeto, abordando práticas de **Design Empático** voltadas à compreensão das necessidades, limitações e experiências dos usuários. O conteúdo foi elaborado com base em princípios de Design Centrado no Usuário (DCU), acessibilidade digital e inclusão.

O Design Empático é uma abordagem que busca compreender profundamente as pessoas que utilizarão um produto ou serviço. No contexto da acessibilidade, significa projetar interfaces e experiências considerando diferentes capacidades físicas, sensoriais, cognitivas e tecnológicas.

O objetivo não é apenas atender requisitos técnicos de acessibilidade, mas também entender como diferentes usuários interagem com sistemas digitais em seus contextos reais.

## Princípios do Design Empático

* [ ] Considerar diferentes perfis de usuários desde o início do projeto.
* [ ] Compreender necessidades, limitações e expectativas dos usuários.
* [ ] Evitar assumir que todos os usuários possuem as mesmas capacidades.
* [ ] Projetar soluções inclusivas para diferentes contextos de uso.
* [ ] Validar decisões de design com usuários reais sempre que possível.
* [ ] Considerar tecnologias assistivas durante todo o processo de desenvolvimento.
* [ ] Promover acessibilidade como requisito do produto e não como etapa final.

## Compreensão dos Usuários

* [ ] Identificar diferentes perfis de usuários.
* [ ] Considerar pessoas com deficiência visual, auditiva, motora e cognitiva.
* [ ] Considerar usuários com limitações temporárias ou situacionais.
* [ ] Mapear objetivos e dificuldades dos usuários.
* [ ] Compreender diferentes níveis de familiaridade tecnológica.
* [ ] Considerar diferentes dispositivos e ambientes de uso.

## Proto-Personas

Proto-personas são representações simplificadas dos principais perfis de usuários esperados para um sistema.

Elas auxiliam equipes a refletirem sobre necessidades específicas antes da realização de pesquisas mais aprofundadas.

### Exemplo 1

#### João

* 68 anos
* Baixa visão
* Utiliza zoom do navegador
* Pouca familiaridade com tecnologia

Necessidades:

* Fontes ampliáveis;
* Alto contraste;
* Navegação simples;
* Conteúdo organizado.

### Exemplo 2

#### Ana

* Usuária de leitor de tela
* Utiliza navegação por teclado

Necessidades:

* Estrutura semântica adequada;
* Textos alternativos;
* Ordem lógica de navegação;
* Indicadores de foco visíveis.

### Checklist para Proto-Personas

* [ ] Criar perfis representativos dos usuários.
* [ ] Documentar objetivos e dificuldades.
* [ ] Identificar tecnologias assistivas utilizadas.
* [ ] Atualizar as proto-personas conforme novas informações surgirem.

## Mapa de Empatia

O mapa de empatia é uma ferramenta utilizada para compreender melhor a experiência do usuário.

Durante sua elaboração, recomenda-se responder às seguintes perguntas:

### O que o usuário vê?

* Interfaces digitais;
* Conteúdo visual;
* Obstáculos de navegação.

### O que o usuário ouve?

* Orientações;
* Feedbacks do sistema;
* Leitores de tela;
* Conteúdos multimídia.

### O que o usuário pensa e sente?

* Frustrações;
* Expectativas;
* Dificuldades;
* Necessidades.

### O que o usuário faz?

* Navega;
* Pesquisa informações;
* Preenche formulários;
* Consome conteúdo.

### Quais são suas dores?

* Barreiras de acessibilidade;
* Linguagem complexa;
* Interfaces confusas;
* Conteúdo inacessível.

### Quais são seus objetivos?

* Concluir tarefas;
* Encontrar informações;
* Consumir conteúdo;
* Utilizar serviços digitais com autonomia.

## Jornada do Usuário

* [ ] Identificar as etapas percorridas pelo usuário.
* [ ] Mapear possíveis barreiras de acessibilidade.
* [ ] Identificar momentos de frustração.
* [ ] Propor melhorias para cada etapa.
* [ ] Validar as soluções propostas.

## Inclusão e Acessibilidade

* [ ] Considerar usuários que utilizam leitores de tela.
* [ ] Considerar usuários que dependem exclusivamente do teclado.
* [ ] Considerar usuários com baixa visão.
* [ ] Considerar usuários com deficiência auditiva.
* [ ] Considerar usuários neurodivergentes.
* [ ] Considerar usuários idosos.
* [ ] Considerar usuários com limitações temporárias.

## Testes com Usuários

* [ ] Realizar testes de usabilidade.
* [ ] Incluir participantes com diferentes perfis.
* [ ] Observar dificuldades encontradas durante a navegação.
* [ ] Registrar problemas recorrentes.
* [ ] Utilizar os resultados para aprimorar o produto.

## Benefícios do Design Empático

* Maior inclusão digital;
* Redução de barreiras de acessibilidade;
* Melhor experiência do usuário;
* Produtos mais fáceis de utilizar;
* Maior conformidade com normas e diretrizes de acessibilidade;
* Desenvolvimento centrado nas necessidades reais das pessoas.
